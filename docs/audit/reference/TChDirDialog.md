- Notable finding: `OkButton` is the one public C++ field that has no direct named Rust counterpart — it is owned entirely by the embedded `Dialog` group by `ViewId` and never stored on `ChDirDialog` itself. This is an intentional deviation (the `Dialog` group owns all children) documented in the type-level doc, but readers coming to find the "ok button" by name may not immediately see it; a doc note on `new` pointing to the delegation model would help.
